# v4 기술스택 결정

이 문서는 일본어 음성 시동 명령 기능의 기술 선택을 기록합니다. v4의 목표는 무료, 빠른 적용, 프론트엔드 중심 구현입니다.

## 선택

| 영역 | 결정 | 이유 |
| --- | --- | --- |
| 음성 인식 | Web Speech API 호환 브라우저 speech recognition | 외부 유료 STT 없이 `ja-JP` 음성 인식을 빠르게 붙일 수 있음 |
| 실행 위치 | 프론트엔드 전용 | 시동 판정은 민감한 전투 판정이 아니며 서버 비용과 키 관리를 늘리지 않음 |
| 언어 설정 | `ja-JP` | 일본어 명령어 인식에 맞춤 |
| 판정 방식 | 키워드 포함 매칭 | MVP에서 자연어 의도 분석 없이도 성공/실패를 안정적으로 분리 |
| fallback | 수동 시동 버튼 | 브라우저 미지원, 권한 거부, 인식 실패 상황에서도 기존 흐름 진입 가능 |
| 저장 | 저장하지 않음 | 음성 원문과 오디오를 backend persistence에 남기지 않음 |

## 유지할 경계

- 시동 명령은 게스트 생성, 로드아웃 이동, 매칭 시작 중 현재 가능한 다음 행동으로만 연결합니다.
- battle action, skill effect, gesture sequence 판정에는 음성 명령을 연결하지 않습니다.
- 음성 transcript는 화면 상태와 테스트에만 사용하고 서버로 전송하지 않습니다.
- 특정 작품 고유 표현은 기본 명령어 세트에 넣지 않습니다.

## 제외

- 억양 평가
- 발음 점수화
- 감정 분석
- 유료 외부 STT 연동
- 완전 오프라인 STT 보장
- 복잡한 자연어 의도 분석

## 기본 명령어 세트

기본 명령:

- `起動して`
- `スタート`
- `始めて`
- `開始`
- `エンジンをかけて`

콘셉트형 범용 명령:

- `結界展開`
- `術式起動`
- `呪力起動`
- `封印解除`
- `開門`
- `解放`
