# v5 구현 순서

## 1. 정보구조 정리

- 왼쪽 탭을 `홈`, `로드아웃`, `연습`, `전적`으로 축소한다.
- `매칭 중`, `전투 중`, `결과`는 탭이 아니라 진행 상태 화면으로 분리한다.
- active battle이나 ended battle이 없을 때 빈 전투/결과 화면으로 이동하지 못하게 한다.

## 2. 홈 CTA 재설계

- 홈은 상태별 primary CTA를 하나만 제공한다.
- 계정 없음: `게스트 시작`.
- 계정 있음, 로드아웃 없음: `로드아웃 설정하기`.
- 계정 있음, 로드아웃 있음: `랭크 1대1 매칭`.
- 큐 대기 중: `큐 취소`.
- 전투 중: `전투로 돌아가기`.
- 결과 확인 가능: `결과 보기`.

## 3. 연습 화면 재설계

- 탭 이름은 짧게 `연습`으로 둔다.
- 화면 제목은 `술식 연습`으로 둔다.
- 카메라 프리뷰를 최상위 시각 요소로 배치한다.
- 연습이 레이팅, 전적, 매칭 로드아웃에 영향을 주지 않는다고 명시한다.
- 연습 중인 술식과 저장된 로드아웃을 분리해서 표시한다.
- 연습 완료 후 `다시 연습`, `로드아웃 저장`, `매칭 시작`을 제공한다.

## 4. 매칭 진행 상태 재설계

- 매칭은 `랭크 1대1 매칭` 버튼으로만 시작한다.
- 매칭 시작 전 계정과 로드아웃 조건을 선제 검증한다.
- 큐 진입 후에는 `매칭 중` 상태 화면으로 전환한다.
- 상태 단계는 `플레이어 준비`, `로드아웃 확인`, `소켓 연결`, `큐 대기`, `상대 발견`, `전투 준비`로 표시한다.
- `SEARCHING` 중에는 `큐 취소`를 제공한다.
- `MATCHED` 이후에는 취소를 잠그고 전투 시작을 기다린다.

## 5. 전역 진행 배너

- 큐 대기 중 다른 목적지로 이동하면 `매칭 대기 중 · 큐 취소` 배너를 유지한다.
- 전투 중 다른 목적지로 이동하면 `전투 진행 중 · 전투로 돌아가기` 배너를 유지한다.
- 결과 확인 가능 상태에서는 `직전 결과 보기` 배너를 선택적으로 제공한다.

## 6. 전투/결과 자동 진입

- `battle.started` 이벤트를 받으면 전투 화면으로 자동 전환한다.
- `battle.ended` 또는 종료 snapshot을 받으면 결과 화면으로 자동 전환한다.
- 결과 화면의 후속 CTA는 `다시 매칭`, `연습`, `전적 보기`, `홈` 순서로 둔다.

## 7. 테스트 보강

- 활성 전투가 없을 때 전투 화면에 직접 진입할 수 없는지 검증한다.
- active result가 없을 때 결과 화면에 직접 진입할 수 없는지 검증한다.
- 로드아웃 미저장 상태에서 매칭 CTA가 잠기고 이유가 표시되는지 검증한다.
- 큐 대기 중 다른 목적지로 이동해도 배너와 큐 취소가 유지되는지 검증한다.
- 연습 완료가 서버 전투 상태, 레이팅, 전적을 변경하지 않는지 검증한다.
- 연습 중인 술식과 저장된 로드아웃이 다를 때 화면에 분리 표시되는지 검증한다.

## 권장 커밋 단위

1. `feat(frontend): 플레이 플로우 내비게이션 정리`
   - 탭 축소, 진행 상태 화면 분리, 홈 CTA 정리.
2. `feat(frontend): 술식 연습 UX 재구성`
   - 카메라 중심 연습 레이아웃, 연습/로드아웃 경계 표시, 완료 CTA.
3. `feat(frontend): 매칭 진행 상태 UX 정리`
   - 단계 표시, 전역 배너, 큐 취소/전투 복귀 흐름.

한 기능당 커밋을 3개 넘기지 않는 원칙에 맞춰 v5는 위 3개 작업 단위로 진행한다.
