# V5 스킬 프리뷰 자산 가이드

## 결론

v5에서는 GIF를 기본 포맷으로 쓰지 않는다.

- 제품 UI 기본 포맷은 `poster + webm + mp4` 조합으로 간다.
- GIF는 문서, PR, 채팅 공유용 파생 산출물로만 제한한다.
- 모든 스킬에 움직이는 자산을 한 번에 붙이지 않는다.
- 먼저 `핵심 스킬 4~6개`만 루프 프리뷰를 만들고, 나머지는 정적 단계 가이드로 시작한다.

## 왜 GIF를 기본으로 두지 않는가

- 용량이 크고 화질 대비 효율이 낮다.
- 모바일과 저속 네트워크에서 로딩 비용이 크다.
- 색 번짐과 계단 현상 때문에 손 모양 디테일이 잘 안 보인다.
- 자막, 로컬라이즈, 속도 조절, 일시정지가 어렵다.
- 이후 자세가 바뀌면 재제작 비용이 높다.

## 권장 자산 구조

각 스킬은 아래 3종만 우선 관리한다.

1. `poster.webp`
짧은 카드 썸네일이나 로딩 전 정지 이미지.

2. `preview.webm`
브라우저용 무음 루프 프리뷰. 기본 재생 포맷.

3. `preview.mp4`
호환성 백업 포맷.

문서 공유가 필요할 때만 아래를 추가한다.

4. `preview.gif`
PR 본문, 노션, 채팅 공유용.

## v5 범위 추천

v5에서는 스킬을 3단계로 나눈다.

1. Hero 스킬
- 홈, 연습, 로드아웃에서 반복 노출되는 대표 스킬
- `poster + webm + mp4` 제공

2. Standard 스킬
- 카드 안에서는 정적 썸네일만 제공
- 상세 가이드에서는 단계 텍스트와 스틸 컷만 제공

3. Experimental 스킬
- 아직 자세가 흔들리거나 인식 기준이 바뀔 가능성이 큰 스킬
- 움직이는 자산 없이 단계 설명만 유지

## 촬영/제작 규칙

- 길이: 2초에서 4초 사이
- 해상도: 640x360 또는 720x405
- 배경: 단색 또는 대비가 분명한 배경
- 프레이밍: 손과 손목이 항상 프레임 안에 남아야 함
- 루프: 시작과 끝이 자연스럽게 이어져야 함
- 텍스트 오버레이: 자산 내부에 굽지 말고 UI에서 얹는다
- 음성: 제외

## 추천 제작 파이프라인

1. 연습 화면에서 기준 자세를 녹화한다.
2. 스킬별 핵심 구간만 2초에서 4초로 트리밍한다.
3. `webm`과 `mp4`를 함께 뽑는다.
4. 대표 프레임을 골라 `poster.webp`를 만든다.
5. 문서 공유가 필요할 때만 `gif`를 추가 변환한다.

## 프론트 메타데이터 권장안

UI 코드는 자산 파일 경로만 알고 있으면 충분하다.

```ts
type SkillPreviewAsset = {
  posterSrc: string;
  loopWebmSrc?: string;
  loopMp4Src?: string;
  docGifSrc?: string;
  status: "hero" | "standard" | "experimental";
};
```

스킬 정의에 바로 큰 바이너리를 묶지 말고, `skillId -> preview asset` 매핑만 별도 관리하는 편이 낫다.

## 화면 반영 우선순위

1. 연습 화면
- 현재 연습 중인 술식 카드에만 프리뷰를 붙인다.

2. 로드아웃 화면
- 선택된 술식 상세 패널에만 프리뷰를 붙인다.

3. 홈 화면
- 대표 CTA 주변에는 1개 이하의 대표 프리뷰만 둔다.

모든 스킬 카드에 동시에 자동재생을 넣는 건 v5에서 피한다.

## 커밋 분리 추천

이 작업도 3커밋 안으로 쪼개는 게 좋다.

1. 프리뷰 메타데이터 구조와 플레이어 컴포넌트 추가
2. Hero 스킬 자산 연결
3. 문서와 폴백 상태 정리

## 최종 권장안

지금은 `GIF를 몇 개 만들지`보다 `어떤 스킬만 움직이는 자산을 가질지`를 먼저 고정하는 게 맞다.

v5 첫 릴리즈는 아래처럼 가는 걸 권장한다.

- 대표 스킬 4~6개만 `webm/mp4/poster` 제공
- 나머지는 단계 텍스트 + 정적 이미지
- GIF는 제품 자산이 아니라 문서 자산으로만 관리
